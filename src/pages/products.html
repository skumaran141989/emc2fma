Project MIRA-J`-I : Service components as product

A software product can be built by pre-developed API services and configuring their communication flow. From my past software industry experience, building various products and features across domains using technologies and conducting case-studies as part of research in EMC2FMA, I have developed a service components suite.

This suite can be used across various domains and very easy to configure and deploy with few hours

Service Components focuses on:

  • Third party API/Web communications
  • Languages tech-depth and usage rationality
  • Notifications enablements
  • Scheduling
  • Data/Distributed processing
  • Datastructures - beyond programming
  • Cloud Storage
